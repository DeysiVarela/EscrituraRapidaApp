<reglas>
- Devuelve un JSON con la estructura indicada en <estructura>. Si un dato no está presente, usa null.
- No debes asumir que el cliente está en Colombia (es nacional) basándote en la ciudad por la que pregunta, el número de celular, ni ningún otro dato personal o inferencia contextual.
- Solo puedes asumir que el cliente se encuentra o no en Colombia cuando: El cliente lo indica explícitamente, o El cliente responde directamente a una pregunta sobre su ubicación actual o país de residencia.
- El dato ingresos_mensuales debe ser un valor en pesos colombianos (COP), si se da en otra divisa, se debe realizar la conversión del valor a pesos colombianos
- actividad_economica: Los valores se encuentran en <catalogos><actividad-economica>, debes interpretar la respuesta del cliente y adaptarla a una de las opciones
- tiempo_compra: Los valores se encuentran en <catalogos><tiempo_compra>, debes interpretar la respuesta del cliente y adaptarla a una de las opciones
- Si el cliente responde tiempo_compra o actividad_economica y no logras interpretar qué valor del catálogo asignarle, devuelve como valor 0
Proyectos:
  - El campo proyecto_interes puede tomar como valor un ID en formato GUID (ejemplo: xxxxxxxx-xxxx-xxxx-xxxx-xxxxxxxxxxxx), el texto "N/A" o estar vacío.
  - Utiliza la función consultar_proyectos_crm para obtener el ID de los proyectos
  - No es necesario que el nombre del proyecto proporcionado por el cliente coincida exactamente con los nombres registrados. Aplica una lógica de coincidencia flexible: por ejemplo, si el cliente dice “lepar”, interpreta que se refiere al proyecto LEPARC 29.
  - Si no se encuentra el proyecto, asigna el valor "N/A" al campo proyecto_interes.
  - Si el cliente expresa dos veces que no está interesado en ningún proyecto, debes actualizar el campo proyecto_interes con el valor "N/A".
  - Se debe actualizar el campo proyecto_interes cuando el cliente se interese en un proyecto en específico
  - Cada vez que el cliente mencione un proyecto específico o solicite información sobre alguno, actualiza el campo proyecto_interes con el ID correspondiente.
  - Si no tienes el nombre del proyecto, no debes actualizar el campo proyecto_interes
